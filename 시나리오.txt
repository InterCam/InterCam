시나리오

1. 메인페이지
- 1위 영상 / 영상 올리기 버튼 / 공지사항 5개만 보이게 / 한개가 뭐였더라

2. 회원가입
- User : 회원가입 Form을 통해 실행
- ADMIN : DB를 통해서만 저장시킬 수 있음
- ANALYST : ADMIN이 직접 지정해줄 수 있음 --> ADMIN에게만 보이는 창(ADMIN 페이지를 만드는게 좋지 않을까?)
2. 로그인 : 실행
- 아이디 찾기 : 이름 & 휴대전화
- 비밀번호 찾기 : 이름 & 휴대전화 & 이메일 --> 이메일에 새로운 비밀번호를 보내고, 해당 이메일로 DB에 저장
                                                          --> 보내는 폼에 비밀번호를 변경할 것을 추천
3. MyPage
- 비밀번호 변경 : 한 번 더 로그인 시행해야 할 수 있음
- 기타 정보 수정 : 그냥 할 수 있음

4. 영상 올리기
- 제목 & 라디오 버튼 선택 & 첨부 파일 올리면 DB에 저장시키는 로직 만들기
- 해당 영상을 리스트에 추가하는 로직 필요
- 내가 올린 영상 리스트만 따로 볼 수 있는 페이지 & 로직 구현
- 내가 올린 영상은 삭제시킬 수 있음

5. 모든 영상 리스트
- 미평가 / 평가 여부 구분 후 View파일에 표시해 줌
- 관리자만 모든 영상을 삭제(수정)시킬 수 있는 권한 : 관리자는 버튼만 보이게
- Pageable(페이징) 사용하여 한 장면에 10페이지 & /page=3을 쉽게 하는 버튼 뷰 파일에서 구현
- 전환 --> 제일 나중에 구현

6. 순위 
- avg_score 중 최고점부터 골라내어 1~9위까지 유튜브 리스트 형식으로 구현
- 링크를 클릭하면 해당 영상으로 바로 가는 것 빼고는 구현 할 것 없음

7. 교수 소개
- ADMIN이 ANALYST를 등록시킬 때 해당 정보를 모두 등록시키므로 그 정보를 그대로 올려줌
- 왼쪽에 모든 교수 리스트가 나오고 클릭하면 해당 교수에 대한 정보가 나오는 것이 좋지 않을까 싶음

8. FAQ / 공지사항 : 모두 구현 완료. View 파일에서 공지사항 추가하는 로직 추가 구현 필요
- FAQ는 DB에서 직접적으로 건드리는게 괜찮을 것 같음
